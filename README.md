# 🚧 Performance Optimization

## 문제
채점 데이터 특성상 데이터가 기하급수적으로 증가

- 채점자 약 3,000명
- 과목 5개, 문제 20개
→ 사용자 1명당 약 100건 데이터 발생

→ 대용량 테이블로 인해 조회 속도 약 3초 이상 발생

---

## 해결
- DB 인덱스 재설계
- 불필요한 JOIN 제거
- SUM / AVG 집계 연산 최적화

---

## 결과
- 조회 속도 개선 (약 3초 → 체감 성능 개선)
- 안정적인 서비스 운영 가능
